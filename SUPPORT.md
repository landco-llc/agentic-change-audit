# Support

English is the canonical version of this policy. The Japanese and Traditional Chinese sections below are provided for convenience and are materially equivalent.

## Support channel

The only support channel for this project is GitHub Issues:

```text
https://github.com/landco-llc/agentic-change-audit/issues
```

Please do not expect support through other channels. This project does not publish a private support email address.

## What is accepted

- Bug reports for the Agentic Change Audit Skill, the Codex Plugin foundation, the validators, and the documentation.
- Reproducible installation problems, with the steps, environment, and observed result.
- Documentation errors, broken links, and translation inconsistencies.

A useful report states what you did, what you expected, and what actually happened.

## Security-sensitive reports

Do not publish secrets, credentials, tokens, private repository contents, or personal data in a public Issue.

If a report cannot be described safely in public, open an Issue that describes only the class of problem and states that details are being withheld, and wait for the maintainers to provide a private channel before sending anything sensitive.

## Response expectations

- Support is **best effort**.
- There is **no guaranteed response time**, and no guarantee of a reply at all.
- Issues and pull requests may be reviewed, deferred, or closed at the maintainers' discretion.
- Submitting an Issue does not guarantee a response, a fix, acceptance, a release, or a maintenance commitment.

## What is not included

The open-source license does not include, and this project does not promise:

- free implementation work or integration work;
- audit services, security certification, or compliance certification;
- legal advice;
- production support, incident response, or uptime commitments;
- commercial or organization-specific support.

Professional implementation support, custom audit-policy design, workflow integration, and organizational adoption may be offered separately as paid services under a separate agreement. They are not part of this repository's license.

### 支援管道

本專案唯一的支援管道為GitHub Issues：

```text
https://github.com/landco-llc/agentic-change-audit/issues
```

本專案不公開私人支援電子郵件地址。

### 受理範圍

- Skill、Codex Plugin基礎、validator與文件的錯誤回報。
- 可重現的安裝問題，並附上步驟、環境與實際結果。

### 涉及安全性的回報

請勿在公開Issue中揭露機密資訊、憑證、token、私有repository內容或個人資料。

若無法在公開場合安全描述，請僅說明問題類型並註明已保留細節，待maintainer提供非公開管道後再傳送敏感內容。

### 回應方式

- 支援為**盡力而為**（best effort）。
- **不保證回應時間**，亦不保證必定回覆。
- Issue與pull request可由maintainer自行斟酌審閱、延後或關閉。
- 提交Issue不代表保證回應、修正、採納、發行或維護承諾。

### 不包含的項目

- 免費的實作或整合工作。
- 稽核服務、資安認證或法遵認證。
- 法律意見。
- 生產環境支援、事件應變或可用率承諾。
- 商業或特定組織專屬的支援。

專業導入支援、稽核政策設計、工作流程整合與組織導入，可能依另行簽署的合約以付費服務個別提供，並不屬於本repository授權的一部分。
