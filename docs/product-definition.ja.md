## 一文での説明

Agentic Change Auditは、ソフトウェア変更をマージまたはリリースする前に監査するための、Evidence-firstかつAgent-neutralな標準および再利用可能なスキルです。

## 目的

本プロジェクトは、AIエージェントおよび人間のレビュー担当者が、ソフトウェア変更を受け入れ、マージし、デプロイし、またはリリースできる状態かを判断できるようにすることを目的とします。

単なるコード品質ではなく、変更全体の妥当性を対象とします。

## 対象利用者

- 個人開発者
- OSSメンテナー
- スタートアップ
- ソフトウェア開発チーム
- AI支援開発者
- コーディングエージェントを利用する非エンジニア
- AI生成または人間作成の変更をレビューする組織

## 対象範囲

- Pull Request
- ローカルのworking tree変更
- リリース候補
- ドキュメント変更
- アプリケーションコード変更
- インフラ変更
- 設定変更
- データベースおよびmigration変更
- 人間が作成した変更
- AIが生成した変更
- 人間とAIによる混合作業

## 対象外

- 自動マージ承認
- 自動本番デプロイ
- 法令または規制への適合認証
- セキュリティ保証
- 人間レビューの代替
- OSS提供に付随する無償サポート

## 中核原則

1. Evidence first
2. Agent neutral
3. 監査対象の同一性を固定する
4. 変更範囲を明示的に管理する
5. 監査後に対象が変更された場合は監査結果を失効させる
6. 停止条件を明確にする
7. 最終判断は人間が責任を持つ

## 中心となる問い

本プロジェクトが問うのは、単に以下ではありません。

> このコードは良いか。

次を問います。

> この変更は、受け入れ、マージ、デプロイ、またはリリースするために、安全かつ十分に検証されているか。

## サポート方針

本OSSプロジェクトは、現状有姿で提供します。

無償の導入支援、トラブルシューティング、実装支援、回答期限の保証は行いません。

IssueおよびPull Requestは、メンテナーの裁量で確認します。

導入支援、カスタム監査ポリシー設計、開発ワークフロー統合、組織導入は、有償業務として提供する場合があります。
