# Release Checklist

[日本語](RELEASE_CHECKLIST.ja.md)

Use this checklist for a manual pre-release or stable release.

The first public release is expected to remain a pre-release until installation and audit behavior have been exercised by external users.

## 1. Fix the release identity

- [ ] Choose a SemVer version without a `v` prefix, for example `0.1.0-rc.1`.
- [ ] Record the exact full `main` commit SHA.
- [ ] Confirm the working tree is clean and `main` equals `origin/main`.
- [ ] Confirm PR and `main` validation workflows are successful.
- [ ] Confirm no unreviewed commit will be included.

## 2. Validate the source

```bash
python -m pip install --disable-pip-version-check \
  -r requirements-validation.txt

python scripts/validate-skill.py .
python -m unittest discover -s tests -p 'test_*.py' -v
```

- [ ] Skill validation passes.
- [ ] All validator and distribution tests pass.
- [ ] `git diff --check` passes.
- [ ] No generated cache, secret, local archive, or temporary file is tracked.

## 3. Generate fixed artifacts

Run the `Package` workflow manually with the intended version, or build locally from the exact release commit.

For a local build, the builder must report `source identity: verified_git_clean`. Do not use `--test-only-unverified-source`; packages marked `unverified_test_fixture` are test data and are not release candidates.

```bash
python scripts/build-distribution.py \
  --version <version> \
  --source-ref <full-source-sha> \
  --output-dir dist/release
```

- [ ] ZIP, external manifest, and SHA256SUMS are produced.
- [ ] The manifest records `source_identity: verified_git_clean`.
- [ ] A second build from the same inputs is byte-identical.
- [ ] The ZIP root is exactly `agentic-change-audit/`.
- [ ] The ZIP contains only the allowlisted runtime files and `PACKAGE-MANIFEST.json`.
- [ ] The package contains no `.github`, `release`, `scripts`, `tests`, cache, or secret file.

## 4. Verify independently

```bash
python scripts/verify-distribution.py \
  dist/release/agentic-change-audit-<version>.zip \
  --manifest dist/release/agentic-change-audit-<version>.manifest.json \
  --checksums dist/release/agentic-change-audit-<version>.SHA256SUMS \
  --expected-version <version> \
  --expected-source-ref <full-source-sha>
```

- [ ] Internal and external manifests are byte-identical.
- [ ] All packaged file sizes and SHA-256 values match.
- [ ] External checksums match the ZIP and manifest.
- [ ] The source repository, source SHA, version, and root are correct.
- [ ] An independent audit records the fixed artifact hashes.
- [ ] Required human release approval is recorded outside the package.

## 5. Prepare the tag and draft release

Create an annotated tag only after the fixed artifacts pass review.

```bash
git tag -a "v<version>" <full-source-sha> \
  -m "Agentic Change Audit v<version>"

git push origin "v<version>"
```

Prepare release notes from [RELEASE_NOTES_TEMPLATE.md](RELEASE_NOTES_TEMPLATE.md).

If the files came from a GitHub Actions artifact download, first extract the outer transport ZIP. The outer transport ZIP is never a GitHub Release asset. Attach exactly the three reviewed inner assets:

```bash
gh release create "v<version>" \
  dist/release/agentic-change-audit-<version>.zip \
  dist/release/agentic-change-audit-<version>.manifest.json \
  dist/release/agentic-change-audit-<version>.SHA256SUMS \
  --verify-tag \
  --draft \
  --prerelease \
  --notes-file <release-notes-file>
```

For a stable release, remove `--prerelease` only after the stability decision is explicitly approved.

- [ ] Tag points to the audited source SHA.
- [ ] Release remains draft while assets are checked.
- [ ] Exactly three inner assets are attached; the outer Actions transport ZIP is not attached.
- [ ] Release notes state the pre-release/stable status and known limitations.
- [ ] No source ZIP generated by this process is presented as the Skill runtime archive.
- [ ] No downloaded GitHub Actions transport ZIP is presented as the Skill runtime archive or a release asset.

## 6. Verify uploaded assets

Download the draft assets into a new empty directory.

- [ ] Downloaded filenames match the reviewed filenames.
- [ ] SHA256SUMS verifies the downloaded ZIP and manifest.
- [ ] `verify-distribution.py` passes against the downloaded assets.
- [ ] Extraction produces `agentic-change-audit/SKILL.md`.
- [ ] The extracted root has no extra version suffix.
- [ ] Claude Code and Codex local installation smoke checks are completed where practical.

## 7. Publish

- [ ] The draft release still points to the audited tag and source SHA.
- [ ] No asset changed after the independent audit.
- [ ] Maintainer approval is explicit.
- [ ] Publish the GitHub Release manually.
- [ ] Reopen and re-audit instead of replacing assets silently if any material change is needed.

## 8. After publication

- [ ] Record the public release URL, tag, source SHA, and three asset hashes.
- [ ] Confirm the release is marked pre-release or stable as approved.
- [ ] Confirm the repository remains clean.
- [ ] Do not promise free installation support, response times, or maintenance.
